# Security + Observability Audit — 2026-05-23

Two parallel reviewers swept the codebase on 2026-05-23 after the
`cleanup/build-warnings` PR landed:

- **Adversarial C++ security review** — focus on memory safety, concurrency,
  lifetime, and attacker-controlled input.
- **Observability / OTel review** — span quality, coverage, propagation,
  attribute hygiene, cost.

This doc captures the findings so they survive past the original session and
the items that aren't getting fixed in this branch have a tracking home.

The `audit/security-and-observability` branch addresses the most-exploitable
items; the rest are still TODO. See the "Status" tag on each item.

---

## Security findings

### Critical

**S-C1 — TLS cert verification disabled in StreamingTTSClient**
`src/server/voice/StreamingTTSClient.cpp:207-233`. `TLS_client_method()` SSL_CTX
is created but never configured with `SSL_CTX_set_verify(SSL_VERIFY_PEER, …)`,
`SSL_CTX_set_default_verify_paths()`, or hostname verification. MITM on the
route to `api.elevenlabs.io` (rogue Wi-Fi, ARP poisoning, etc.) succeeds and
the ElevenLabs API key flows over the attacker's link in the WebSocket upgrade
header and the BOS message body. **Status: fixed in this branch.**

**S-C2 — WebSocket frame `payloadLen` unbounded**
`src/server/voice/StreamingTTSClient.cpp:363-394`. `payloadLen` read from the
network is used directly in `std::vector<uint8_t>(payloadLen)`. A `uint64_t`
length claiming 2^63 bytes crashes the process with `bad_alloc`; smaller-but-
still-large values let an attacker slowly OOM. The fragmented-frame path
(line 413) accumulates into `fragmentBuffer` with no cap either. **Status:
fixed in this branch.**

**S-C3 — Shell injection in popen call sites**
`src/server/voice/LipSyncProcessor.cpp:221,226,231`,
`src/server/voice/AudioConverter.cpp:98-99,174,278-279`,
`src/server/ws/service/SoundService.cpp:485-499`. Every `popen` builds its
command via `fmt::format("…\"{}\"…", path)`. The lipsync REST endpoint
(`POST /api/v1/sound/generate-lipsync`) takes `requestBody->sound_file`
unsanitized into this path. Animation metadata stored in MongoDB also flows
in (anyone with DB write → RCE). **Status: fixed in this branch — argv-based
`runChildProcess` helper.**

**S-C4 — E1.31 galaxy map race + detached worker**
`lib/e131_service/src/E131Server.cpp`. `galaxy` (`std::unordered_map`) is
iterated by the worker thread every 20 ms while the event-loop thread can
`galaxy[universe] = …`. Concurrent insert during iteration is UB →
shared_ptr<Universe> double-free / torn reads possible. Triggered by any
client posting an animation on a previously-unseen universe. Separately,
`worker.detach()` then `worker.join()` in shutdown: `joinable()` is false
after detach, so shutdown closes the socket while the worker is still
running. **Status: fixed in this branch — `std::shared_mutex` + owned
thread.**

### High

**S-H1 — Vector race in `StreamingAdHocSession`**
`src/server/voice/StreamingAdHocSession.cpp:309, 244`. `addText` pushes
into `offsetFutures_` / `requestIdFutures_` under `offsetMutex_`. The async
worker reads `offsetFutures_[N-2]` and `requestIdFutures_[N-2]` without the
lock. Concurrent `addText` can reallocate the vector → dangling indexing.
**Status: fixed in this branch.**

**S-H2 — `mouth_slot` out-of-bounds write**
`src/server/voice/StreamingAdHocSession.cpp:319-326`. `frameData[mouthSlot] = …`
with no bounds check; `mouthSlot` is creature-config controlled, `frameData`
size is animation controlled. **Status: fixed in this branch.**

**S-H3 — Inbound WebSocket message has no cap**
`src/server/ws/websocket/ClientConnection.cpp:151`. `m_messageBuffer.writeSimple`
accumulates multi-frame WebSocket payload with no upper bound. Multi-GB
inbound payload → OOM. **Status: fixed in this branch.**

**S-H4 — spdlog in signal handler**
`src/server/main.cpp:202-217`. `signal_handler` calls `info(...)` →
mutexed sinks → libfmt → not async-signal-safe. If SIGINT lands while the
main thread holds the sink mutex, the handler deadlocks and only SIGKILL
shuts the process down. **Status: fixed in this branch.**

**S-H5 — `thread_local mongocxx::pool::entry`**
`src/server/database.cpp:46`. Every oatpp worker thread permanently holds a
MongoDB pool entry for its lifetime. Pool exhaustion under sustained load
hangs/throws on `acquire()`. **Status: NOT FIXED — needs design (per-call
acquire changes the contract for callers using the returned `collection`).**

**S-H6 — `CreatureRuntimeDto` mutation races**
`src/server/ws/service/CreatureService.cpp:765-797`. `getOrCreateRuntime`
takes `runtimeMutex` for the lookup but `setActivityState` mutates DTO
fields outside the lock. Concurrent calls from event loop, HTTP handlers,
and fixture activity races. **Status: NOT FIXED — needs copy-modify-publish
design.**

### Medium

- **S-M1 — non-thread-safe `std::gmtime`** in `database.cpp:236`,
  `helpers.cpp:228,236`, `CreatureLogSink.h:70`. Use `gmtime_r`.
  **Status: NOT FIXED.**
- **S-M2 — `frameCount` non-atomic** in `eventloop.h:37`. Read from
  worker/handler threads. Works on x86_64 by luck. **Status: NOT FIXED.**
- **S-M3 — `DMXEvent::channelOffset` u32→u16 silent narrowing**
  in `eventloop/events/dmx.cpp:33`. **Status: NOT FIXED.**
- **S-M4 — `hasStarted_` non-atomic** in `PlaybackSession.h:182-191` and
  `StreamingPlaybackSession.h:144-153`. Currently only called from event
  loop so safe, but API allows misuse. **Status: NOT FIXED.**
- **S-M5 — `ClientConnection::ourSocket` reference** outliving the
  underlying oatpp socket would be UAF. Catch-all in `sendTextMessage`
  hides corruption. **Status: NOT FIXED.**
- **S-M6 — `contains()` + `get()` TOCTOU** in
  `LegacyAnimationScheduler.cpp:195-196`, `playback-runner.cpp:354-355`,
  `streaming-playback-runner.cpp:164-165`. Cache is append-only today,
  so latent. Use `tryGet`. **Status: NOT FIXED.**
- **S-M7 — `response.find("101")`** in `StreamingTTSClient.cpp:288` matches
  any "101" anywhere in HTTP response, not just the status line.
  **Status: NOT FIXED — low actual impact, but cosmetically wrong.**

### Low

- **S-L1** — `std::random_device` only 32-bit seeds the `mt19937` for
  WebSocket masking. Fine for RFC purpose. **Status: NOT FIXED (won't fix).**
- **S-L2** — API key in BOS JSON body (`StreamingTTSClient.cpp:614`).
  If logged at trace, leaks. **Status: NOT FIXED.**
- **S-L3** — `ObjectCache::get(key)` throws on miss. Sweep remaining
  callers to `tryGet`. **Status: NOT FIXED.**

### Healthy patterns observed

- `ObjectCache` shared_mutex + tryGet design is correct.
- `Universe::setFragment` overflow-checks `firstSlot + N`.
- `AudioStreamBuffer::loadWaveFile` does `SIZE_MAX / samplesPerFrame`
  overflow guard before computing frame counts.
- `OpusEncoderWrapper` mutex + RAII discipline is consistent.
- `SessionManager::interrupt` holds the lock through the entire
  cancel→schedule→register sequence (per the comment, this was actively
  reasoned about and fixed).
- `SoundController::sanitizeFilename` is strict regex + path traversal
  + null-byte check. The bug is that other endpoints don't call it.

## Observability findings

### High impact

**O-H1 — Five controllers create no request spans**
`PlaylistController.h`, `VoiceController.h`, `DebugController.h`,
`MetricsController.h`, `StaticController.h`. Service spans downstream
become orphan roots with no HTTP context. AGENTS.md already calls for a
`runEndpoint(...)` helper. **Status: in progress this branch.**

**O-H2 — `setHttpStatus(200)` outside try/catch**
Recurring pattern across `CreatureController`, `AnimationController`,
`SoundController`, `SpeechToTextController`, `StreamingAdHocController`.
When the service throws, the RequestSpan dtor defaults to 200 →
`http.status_code = 200` for what was actually a 4xx/5xx.
`DmxFixtureController` shows the fix (`withSpanStatus(...)`).
**Status: in progress this branch via runEndpoint helper.**

**O-H3 — `error.type` recorded as boolean**
Sticky-bool column in Honeycomb. ~40+ writes across `StreamFrameHandler`,
`SensorReportHandler`, `AnimationController`, `SoundController`,
`StreamingAdHocController`. `GROUP BY error.type` returns "true" instead
of the actual classification string. Per AGENTS.md, fix is Honeycomb-side
(drop and re-type) or rename the field project-wide. **Status: NOT FIXED
— needs Honeycomb-side decision.**

**O-H4 — `StreamFrameHandler` 50 Hz path dereferences span unchecked**
`src/server/ws/messaging/StreamFrameHandler.cpp:112-126`. If
`createSamplingSpan` returns nullptr (which it does when `!initialized_`),
every frame crashes the WebSocket thread. Also a crash bug, not just an
observability concern. **Status: fixed in this branch.**

**O-H5 — Uninstrumented hot paths**
`DMXEvent::execute()`, `MultiOpusRtpServer::send`, GPIO/status-light
events, cache-invalidate events, `SessionManager::cancel`/`setActivityState`,
WebSocket broadcast helpers. **Status: NOT FIXED — separate
instrumentation pass.**

### Quality

**O-Q1 — `MusicEvent` orphan root span**
`src/server/eventloop/events/music.cpp:146-148, 289-291` creates a new
"music_event" root span as fallback when `!parentSpan`, but the caller
always provides one. Dead code → orphan roots. **Status: NOT FIXED.**

**O-Q2 — `SamplingSpan::setError` mislabels late-reified spans**
`src/util/ObservabilityManager.cpp:1062, 1085, 1127` hardcodes
`"eventloop.frame"` for the lazily created span name. Any other
SamplingSpan user (StreamFrameHandler, playback-runner, etc.) gets
mislabeled on error. **Status: fixed in this branch.**

**O-Q3 — `StreamingAdHocSession.sentence.{N}` span name cardinality**
`StreamingAdHocSession.cpp:233` embeds unbounded `N` into the span name.
**Status: fixed in this branch.**

**O-Q4 — Last-wins attribute overwrites in playback runner**
`playback-runner.cpp:283-403` iterates tracks and overwrites
`creature.id` / `fixture.id` / `track.kind` each iteration. Use span
events per track or aggregate counters instead. **Status: NOT FIXED.**

### Context propagation

**O-P1 — `StreamingTimeoutEvent` loses parent context**
`StreamFrameHandler.cpp:96-100` passes nullptr to `setActivityState`
when arming/firing the timeout. Stash a traceparent / link at arm time
like `FixturePatternRunner::ActivePattern` does. **Status: NOT FIXED.**

**O-P2 — `streaming-playback-runner.cpp:167` `db->getCreature(creatureId, nullptr)`**
should pass session span. **Status: NOT FIXED — trivial.**

**O-P3 — `playback-runner.cpp:246-247` schedules StatusLightEvent without
context**. **Status: NOT FIXED — depends on O-H5.**

**O-P4 — `JobWorker` passes `nullptr` to `sessionManager->interrupt`**
`JobWorker.cpp:884`. Should pass `jobState.span`. **Status: NOT FIXED.**

**O-P5 — `StreamingAdHocSession::playbackThreadFunc` passes no parent**
to `sessionManager->queueAnimation` / `interrupt` (line 434, 440).
**Status: NOT FIXED.**

**O-P6 — `CooperativeAnimationScheduler::scheduleAnimation` creates root**
at `CooperativeAnimationScheduler.cpp:62` even though `playlist.cpp:316-318`
has a `PlaylistEvent` span. **Status: NOT FIXED.**

### Attribute hygiene

**O-A1 — Sensitive text content on spans**
`JobWorker.cpp:555` (speech.text), `JobManager.cpp:31` (job.details),
`StreamingAdHocSession.cpp:238` (sentence.text), `WhisperLipSyncProcessor.cpp:530`
(transcript.text). Drop, truncate, or hash. Length already recorded.
**Status: fixed in this branch.**

**O-A2 — `LipSyncProcessor.cpp:349 rhubarb.command`**
Records full shell command (with local absolute paths). Replace with
`rhubarb.input_file` (basename) + separate `rhubarb.binary_path`.
**Status: addressed in this branch as part of S-C3 popen → posix_spawn
work.**

**O-A3 — Mixed naming conventions**
`file_path` vs `sound.file` / `sound.path`; `active_universe` vs
`universe` vs `fixture.universe`; `phase` used with two different
semantics. Pick one convention; rename. **Status: NOT FIXED.**

**O-A4 — Redundant `success` / `result.success` booleans**
`AnimationController.h:305, 575`. Span status already conveys this.
**Status: NOT FIXED.**

**O-A5 — `component = "creature-server"` duplicates `service.name`**
`ObservabilityManager.cpp:727, 1027`. Drop. **Status: NOT FIXED.**

**O-A6 — Copy-paste in StreamFrameHandler**
`StreamFrameHandler.cpp:152` catch-all sets `error.type = "std::bad_cast"`
which is wrong for the `catch(...)`. **Status: fixed as part of O-H4 in
this branch.**

### Cost / cardinality

- **O-C1** — `StreamFrameHandler::processMessage` + `SensorReportHandler` use
  default 0.001 sampling rate; AGENTS.md gold standard is 0.0005.
  **Status: NOT FIXED.**
- **O-C2** — `rhubarb.command` / `file_path` jobId-keyed paths fragment.
  **Status: addressed via S-C3.**
- **O-C3** — `runner.frame` / `runner.next_frame` / `frame.number` int64
  per-frame values. Expected and useful. **Status: keep, no fix needed.**

### Healthy patterns observed

- `RequestSpan` / `OperationSpan` / `SamplingSpan` RAII abstraction with
  auto-status, `const char*` overload preventing silent `bool` decay.
- W3C traceparent propagation implemented correctly from scratch.
- Creature subsystem is exemplary (every controller → service → DB call has
  spans with proper parent chains).
- `SamplingSpan` design (sampled normal / always-export on error / lazy
  reification) is right for the 1ms event loop.
- Error paths consistently call `setError(message)` with descriptive text,
  and catch sites call `recordException(e)`.
